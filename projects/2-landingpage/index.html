Core Features

Everything your team needs to ship

Built on Firebase with real-time sync and clean BLoC architecture — so your data is always fresh and your code always clean.

🔥

Real-Time Collaboration

Powered by Firebase Firestore streams. Every task update, priority change, and status toggle syncs instantly across all connected devices — no refresh required.

🗂️

Project Management

Organize work into color-coded projects with progress indicators. Create, edit, and archive projects with full CRUD support and visual completion tracking.

🔐

Secure Authentication

Firebase Auth with email/password login, persistent sessions, and auto-login. Every user sees only their own workspace — clean and private by design.

⚡

BLoC State Management

Built with flutter_bloc 8.x and clean architecture. Predictable state transitions, testable BLoCs, and separated concerns from data layer to UI.

🌓

Dark & Light Theme

Full dark/light mode powered by ThemeCubit with hydrated_bloc for persistence. Theme preference survives app restarts — no flicker on launch.

📊

Task Prioritization

Low, Medium, and High priority levels with visual color coding. Filter tasks by status — All, Active, or Completed — to keep focus where it matters.

Under the Hood

Engineered for scale & clarity

A portfolio-grade architecture that teaches as much as it ships.

Clean Architecture Layers

Domain, Data, and Presentation layers are fully separated. Repositories abstract Firebase, keeping business logic framework-agnostic.

Stream-Driven Task Updates

TaskBloc integrates Firestore StreamSubscriptions so UI stays in sync with the server — no polling, no stale data.

Dependency Injection via GetIt

Service locator pattern wires repositories and data sources at startup — making the codebase testable and modular.

Form Validation with BLoC

Login and register forms use dedicated BLoC patterns with inline error states — real-time validation before any Firebase call is made.

Due Dates & Completion Status

Every task supports a due date and toggle completion. Overdue tasks surface automatically with clear visual indicators.

Workspace-Level Organization

Each user gets a default workspace. Projects live inside workspaces — a clean hierarchy that scales naturally to team use cases.
